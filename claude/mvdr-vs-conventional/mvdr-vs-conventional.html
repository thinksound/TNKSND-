How MVDR works — and how it contrasts with traditional beamformers

Conventional (delay-and-sum / "traditional")

Weights point straight at the look direction and never change: w = a(θ₀) / M. The pattern depends only on array geometry — fixed mainlobe, fixed ~−13 dB sidelobes for a uniform array. It is data-independent: it has no idea where the interference is, so it cannot null it. It is optimal only for spatially white noise.

MVDR / Capon (adaptive)

Minimise the total output power subject to a distortionless constraint — unity gain toward the look direction: min wᴴRw s.t. wᴴa(θ₀)=1  ⟹  w = R⁻¹a(θ₀) / (a(θ₀)ᴴR⁻¹a(θ₀)). Because R (the spatial covariance) contains the interference, minimising power forces deep nulls onto the interferers while protecting the target. It is data-dependent and optimal in SINR.

The key relationship

Set R = σ²I (white noise, no interferers — i.e. crank the loading slider up) and the MVDR solution collapses exactly to w = a(θ₀)/M — the conventional beamformer. So conventional beamforming is just MVDR's special case when nothing is worth nulling. The cost of MVDR's adaptivity is white-noise gain: aggressive nulling can amplify uncorrelated sensor noise (see the WNG row), which is why diagonal loading (the noise slider) is used to trade nulling depth for robustness.

MVDR の仕組み — 従来型ビームフォーマとの違い

従来型（遅延和 /「トラディショナル」）

重みはルック方向をまっすぐ向き、決して変化しません： w = a(θ₀) / M。パターンはアレイ形状のみに依存 — 固定のメインローブ、均一アレイでは約 −13 dB の固定サイドローブ。データ非依存で、 干渉源の位置を知らないため打ち消せません。空間的に白色な雑音に対してのみ最適です。

MVDR / Capon（適応型）

無歪拘束（ルック方向に利得1）のもとで出力総電力を最小化： min wᴴRw s.t. wᴴa(θ₀)=1  ⟹  w = R⁻¹a(θ₀) / (a(θ₀)ᴴR⁻¹a(θ₀))。 R（空間共分散）が干渉を含むため、電力最小化はターゲットを保ちつつ 干渉源に深いヌルを強制します。データ依存で、SINR の意味で最適です。

重要な関係

R = σ²I（白色雑音・干渉源なし — ローディングのスライダーを上げる）とすると、MVDR 解はちょうど w = a(θ₀)/M（従来型ビームフォーマ）に一致します。つまり従来型は、打ち消すべきものが無いときの MVDR の特別な場合です。MVDR の適応性の代償は白色雑音利得：積極的なヌル形成は無相関なセンサ雑音を 増幅し得ます（WNG 行を参照）。そのため対角ローディング（雑音スライダー）でヌルの深さと頑健性をトレードオフします。
