How to use this across the 2 weeks2週間でこのツールをどう使うか

Do 3 problems per day and talk out loud the whole time as if an interviewer is in the room: first verbalize which pattern the prompt cues, restate the brute force, then the optimal, state your time/space complexity before coding, and narrate edge cases as you implement. Start each session in the Pattern Deck to reload templates from memory, then run Drill Mode with the per-problem timer (aim < 25 min) and self-rate honestly so spaced repetition resurfaces your weak patterns. Use the 25-min countdown for one timed problem each day to simulate loop pressure, and let the Checklist progress bar tell you which patterns you've under-covered. Quality of out-loud reasoning > volume — at E7 the signal is structured communication, not LeetCode count.

1日3問を解き、面接官が同席しているつもりで常に声に出して進めてください。まず設問がどのパターンを示唆しているかを言葉にし、ブルートフォース解を述べてから最適解へ移り、コーディング前に時間・空間計算量を宣言し、実装しながらエッジケースを口頭で説明します。各セッションはまず Pattern Deck でテンプレートを記憶から再現し、続いて問題ごとのタイマー付き Drill Mode（目標 < 25分）を行い、間隔反復が苦手なパターンを再浮上させるよう正直に自己採点してください。1日1問は 25分カウントダウン を使い本番ループの時間的プレッシャーを再現し、Checklist の進捗バーで手薄なパターンを把握しましょう。声に出した推論の質 > 量です。E7 で見られるのは構造化されたコミュニケーションであり、LeetCode の解いた数ではありません。
